# 后端架构设计说明文档

## 后端技术栈

- **框架**：FastAPI
- **Web服务器**：Uvicorn
- **数据库ORM**：SQLAlchemy
- **数据库**：SQLite（本地）/ MySQL 8.0+（生产）
- **认证**：JWT (PyJWT) + Bcrypt (passlib)
- **日志**：Python Logging + RotatingFileHandler
- **测试**：Pytest + TestClient
- **部署**：Docker + Docker Compose
- **文档**：Swagger/OpenAPI (自动生成)

## 配置管理 (config.py)

支持多数据库模式切换：

```python
class Settings(BaseSettings):
    DATABASE_MODE: str = "sqlite"        # sqlite | mysql
    SQLITE_URL: str = "sqlite:///./radiomanager.db"
    DATABASE_URL: str = "mysql+pymysql://..."

    @property
    def ACTIVE_DATABASE_URL(self) -> str:
        if self.DATABASE_MODE == "sqlite":
            return self.SQLITE_URL
        return self.DATABASE_URL
```

## 认证系统

使用 JWT Bearer Token 认证：

- 令牌类型：HS256
- 过期时间：AccessToken 7天 / RefreshToken 30天
- 密码加密：bcrypt (passlib)
- 认证依赖：`get_current_user()`（从HTTP Bearer头解析token）

## 数据库引擎

根据 `DATABASE_MODE` 自动选择引擎参数：

- **SQLite**：`check_same_thread=False`，启用 WAL 模式和 foreign_keys
- **MySQL**：连接池 `pool_pre_ping=True`，`pool_recycle=3600`

## 台站与位置设计

Station（台站）和 Location（位置）分离：

- **Station** 仅包含 `callsign`（呼号），是日志的归属单位
- **Location** 包含电台型号、天线、梅登网格、QTH 等详情
- 一个 Station 可以拥有多个 Location
- 同一用户全局只能激活一个 Location
- 新建日志时自动通过激活的 Location 找到其所属 Station

## 业务逻辑服务

### 日志服务 (LogService)
- `create_log`：自动通过激活位置填充 `station_id`
- `get_logs`：支持 `station_id` 过滤
- `get_stats`：通联统计计算

### 位置服务 (LocationService)
- `activate_location`：取消用户所有其他位置的激活状态
- 首个位置自动激活

### 呼号查询服务 (CallsignService)
- 优先查询本地缓存
- 缓存未命中时调用 QRZ.com XML API
- 结果自动写入缓存

### 导入导出服务 (ImportExportService)
- `import_adi`：解析ADI文件，逐条写入日志
- `export_adi`：按台站+日期过滤导出
- 导出文件命名格式：`{日期}_{台站呼号}.adi`

## API路由

| 前缀 | 模块 | 主要端点 |
|------|------|---------|
| `/api/v1/auth` | 认证 | register, login, me, change-password |
| `/api/v1/users` | 用户 | me |
| `/api/v1/logs` | 日志 | CRUD + import + export + stats/overview |
| `/api/v1/stations` | 台站 | CRUD |
| `/api/v1/locations` | 位置 | CRUD + activate + active/current |
| `/api/v1/stats` | 统计 | overview |
| `/api/v1/callsigns` | 呼号 | lookup, batch, search, cache |
| `/api/v1/sync` | 同步 | github config/push/pull, history |
| `/api/v1/shortcuts` | 快捷链接 | 列表 + CRUD |

## 中间件

1. **LoggingMiddleware** — 记录每个请求的方法、路径、状态码、耗时
2. **ErrorHandlerMiddleware** — 捕获所有未处理异常，返回统一 JSON 错误响应
3. **RateLimitMiddleware** — 基于内存的 IP 速率限制（默认 100请求/分钟）

## 测试

使用 Pytest + FastAPI TestClient：

```bash
cd backend
pytest                    # 运行所有测试
pytest --cov=app          # 带覆盖率
pytest tests/test_auth.py # 单文件测试
```

测试数据库使用 SQLite 内存模式，每次测试自动建表和清理。

## 部署

### 本地部署（SQLite）
```bash
python -m app.scripts.init_db   # 创建表 + 测试账号
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### Docker部署（MySQL）
```bash
docker-compose up -d
docker-compose exec backend python -m app.scripts.init_db
```
