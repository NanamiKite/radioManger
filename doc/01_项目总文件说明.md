# RadioManager - 业余无线电日志管理工具

## 项目概述

RadioManager是一款基于FastAPI + Vue3的跨平台业余无线电日志管理工具。提供日志管理、台站与位置管理、导入导出、呼号查询等功能，支持多种部署方案（本地SQLite、局域网MySQL、云服务器）。

## 项目特性

- ✅ **多部署方案**：SQLite纯本地部署 / Docker局域网部署 / 云服务器部署
- ✅ **台站+位置树状管理**：一个台站对应一个呼号，可拥有多个位置（Home/Mobile/Field Day），同一用户只能激活一个位置
- ✅ **完整日志管理**：增删改查、导入导出（ADI/ADIF）、台站维度过滤
- ✅ **呼号查询**：集成QRZ.com查询功能，支持本地缓存
- ✅ **DX Cluster 实时 Spot**：Telnet 连接公共 DX Cluster 节点，实时接收 spot 并通过 WebSocket 推送前端
- ✅ **回收站**：日志删除后保留 7 天可恢复，台站删除时其日志自动进入回收站
- ✅ **用户权限管理**：用户认证、JWT令牌、数据隐私
- ✅ **国际化**：支持中文和英文
- ✅ **Docker部署**：支持容器化

## 技术栈

### 前端
- **框架**：Vue 3 + TypeScript
- **构建工具**：Vite
- **桌面应用**：Electron（可选）
- **UI框架**：Element Plus
- **状态管理**：Pinia
- **HTTP客户端**：Axios
- **路由**：Vue Router
- **国际化**：vue-i18n

### 后端
- **框架**：FastAPI (Python 3.10+)
- **ORM**：SQLAlchemy
- **数据库**：SQLite（本地开发）/ MySQL 8.0+（生产）
- **认证**：JWT (PyJWT) + Bcrypt
- **数据验证**：Pydantic V2
- **日志**：Python logging + RotatingFileHandler

### DevOps
- **容器化**：Docker + Docker Compose
- **版本控制**：Git + GitHub

## 部署模式

### 1. 纯本地部署（默认）
- 数据库：SQLite（无需安装MySQL）
- 启动：`./start-local.sh`
- 适用：个人使用、离线环境

### 2. 局域网部署
- 数据库：MySQL（Docker容器）
- 启动：`docker-compose up -d`
- 适用：小型俱乐部

### 3. 云服务器部署
- 数据库：MySQL + Redis
- 前端 Nginx + SSL
- 适用：多用户生产环境

## 核心模块

### 日志管理模块
- 日志展示（表格视图，台站维度过滤）
- 日志编辑（新增、编辑、删除）
- 日志导入（支持.adi/.adif格式，批量提交500条/批，去重）
- 日志导出（按台站+日期范围导出，ADI格式兼容 WSJT-X）
- 回收站（软删除日志保留7天可恢复，台站删除时其日志自动进入回收站）

### 用户认证模块
- 用户注册（用户名+邮箱+密码）
- 用户登录（JWT令牌认证）
- 时区与语言偏好设置

### 台站与位置管理模块
- 台站：仅包含呼号标识，软删除后同名重建可恢复
- 位置：电台型号、天线、梅登网格、QTH
- 树状结构展示，单选激活一个位置
- 级联删除：删除台站时其日志进回收站，位置软删除

### 呼号查询模块
- QRZ.com集成查询（缓存优先，30天过期回退旧缓存）
- 离线DXCC前缀推断（500+前缀模式匹配）
- 查询结果缓存
- 缓存/实时/离线标记

### DX Cluster 模块
- Telnet连接公共DX Cluster节点（8个预设节点：VE7CC/K1TTT/W3LPL等）
- 实时接收并解析DX spot（正则解析 + 频率→波段转换）
- WebSocket实时推送前端（历史spot一次性推送 + 持续实时推送）
- 连接状态管理（连接/断开/超时处理，不自动重连）

### 统计分析模块
- 总通联数、DXCC国家数
- QSL/eQSL/LOTW统计
- 距离统计
- 波段/模式/DXCC维度统计

### 数据同步模块（预留）
- GitHub集成（Pull/Push）
- 同步历史记录

## 环境变量

参考 `backend/.env.example`。关键变量：

| 变量 | 说明 |
|------|------|
| `DATABASE_MODE` | `sqlite`（默认）或 `mysql` |
| `SQLITE_URL` | SQLite连接串 |
| `DATABASE_URL` | MySQL连接串 |
| `SECRET_KEY` | JWT密钥 |
| `ENABLE_TEST_ACCOUNT` | 是否自动创建测试账号 |

## 安全设计

- **密码加密**：bcrypt加密存储
- **传输安全**：HTTPS/TLS加密
- **数据隐私**：用户日志数据仅当前用户可见
- **配置管理**：环境变量管理敏感配置
- **API认证**：JWT令牌认证
- **CORS策略**：明确的跨域资源共享策略
