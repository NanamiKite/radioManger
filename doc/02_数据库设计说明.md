# 数据库设计说明文档

## 概述

RadioManager支持两种数据库后端，通过 `DATABASE_MODE` 环境变量切换：

| 模式 | 数据库 | 适用场景 |
|------|--------|---------|
| `sqlite`（默认） | SQLite (WAL模式) | 本地开发、个人使用 |
| `mysql` | MySQL 8.0+ | 局域网/云服务器生产部署 |

ORM框架为 SQLAlchemy，迁移工具为 Alembic（可选）。

## 数据库架构

### 核心数据表（共9张）

1. **users** — 用户账户
2. **stations** — 台站（仅呼号）
3. **locations** — 台站位置（含激活状态）
4. **qso_logs** — 通联日志
5. **callsign_cache** — 呼号查询缓存
6. **statistics** — 统计数据
7. **log_files** — 导入文件追踪
8. **sync_history** — 同步历史
9. **deleted_logs** — 删除日志恢复

### 表设计原则
1. 所有表包含创建时间和更新时间
2. 主键使用自增 INT
3. 外键关系使用 ON DELETE CASCADE
4. 关键字段创建索引以优化查询性能
5. 使用逻辑删除（soft delete）保留数据追踪

---

### 1. users（用户表）

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(100),
    avatar_url VARCHAR(500),
    role VARCHAR(10) DEFAULT 'user',  -- user / admin
    timezone VARCHAR(50) DEFAULT 'UTC',
    language VARCHAR(10) DEFAULT 'zh-CN',
    is_active INTEGER DEFAULT 1,
    is_deleted INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

---

### 2. stations（台站表）

仅包含台站呼号标识，详细配置信息在 `locations` 表中。

```sql
CREATE TABLE stations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    callsign VARCHAR(20) NOT NULL,
    is_deleted INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, callsign)
);
```

---

### 3. locations（位置表）

一个台站可以有多个位置，同一用户全局只能激活一个位置。

```sql
CREATE TABLE locations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    station_id INTEGER NOT NULL REFERENCES stations(id),
    name VARCHAR(100) NOT NULL,           -- 位置名称 (Home/Mobile/Field)
    grid_square VARCHAR(6),               -- 梅登网格
    radio_model VARCHAR(100),             -- 电台型号
    antenna_model VARCHAR(100),           -- 天线型号
    antenna_height DECIMAL(5,2),          -- 天线高度(米)
    qth VARCHAR(200),                     -- QTH描述
    is_active INTEGER DEFAULT 0,          -- 全局唯一激活标记
    is_deleted INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE CASCADE
);
```

---

### 4. qso_logs（通联日志表）

核心的通联记录表，通过 `station_id` 关联到台站，通过 `location_id` 关联到操作位置。

```sql
CREATE TABLE qso_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    station_id INTEGER NOT NULL REFERENCES stations(id),
    location_id INTEGER REFERENCES locations(id) ON DELETE SET NULL,

    -- 基本通联信息
    call_sign VARCHAR(20) NOT NULL,
    qso_date DATE NOT NULL,
    qso_date_off DATE,
    time_on TIME,
    time_off TIME,

    -- 频率和模式
    freq DECIMAL(10,6),
    freq_rx DECIMAL(10,6),
    band VARCHAR(10),
    band_rx VARCHAR(10),
    mode VARCHAR(20),

    -- 信号报告
    rst_sent VARCHAR(5),
    rst_rcvd VARCHAR(5),

    -- 对方信息
    grid_square VARCHAR(6),
    operator VARCHAR(50),
    qth VARCHAR(200),

    -- QSL状态
    qsl_sent VARCHAR(1) DEFAULT 'N',   -- Y/N/R/I
    qsl_rcvd VARCHAR(1) DEFAULT 'N',
    eqsl_sent VARCHAR(1) DEFAULT 'N',
    eqsl_rcvd VARCHAR(1) DEFAULT 'N',
    lotw_sent VARCHAR(1) DEFAULT 'N',
    lotw_rcvd VARCHAR(1) DEFAULT 'N',

    -- 扩展信息
    distance INTEGER,
    dxcc VARCHAR(100),                   -- DXCC实体名称（自动推断）
    tx_pwr INTEGER,                      -- 发射功率(W)
    my_gridsquare VARCHAR(6),            -- 操作员梅登网格（从激活位置自动填入）
    station_callsign VARCHAR(20),        -- 台站呼号（WSJT-X导出字段）
    comment TEXT,
    is_deleted INTEGER DEFAULT 0,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (station_id) REFERENCES stations(id) ON DELETE RESTRICT,
    FOREIGN KEY (location_id) REFERENCES locations(id) ON DELETE SET NULL
);
```

**关键索引**：(user_id, qso_date)、(call_sign)、(band)、(mode)、(location_id)

**QSL联动逻辑**：当 `qsl_rcvd='Y'` 时，自动设置 `qsl_sent='Y'`、`lotw_sent='Y'`、`lotw_rcvd='Y'`、`eqsl_sent='Y'`、`eqsl_rcvd='Y'`。

**自动填充**：创建日志时若未指定 `station_id`，系统通过当前激活位置自动推断 `station_id` 和 `location_id`，并自动填入 `my_gridsquare`、`dxcc`（基于呼号前缀）、UTC 时间。

### 8. sync_history（同步历史表）

记录与GitHub或云服务的同步操作。

| 字段 | 说明 |
|------|------|
| sync_type | push/pull/merge |
| source | github/cloud |
| status | pending/success/failed |
| added/updated/deleted_count | 操作统计 |
| conflict_count | 冲突数 |

---

### 9. deleted_logs（删除日志表）

保存被删除的日志完整数据用于恢复审计。**保留期 7 天**，过期后由 `cleanup_expired` 自动清理。

| 字段 | 说明 |
|------|------|
| log_id | 原日志ID |
| log_data | JSON格式完整备份 |
| delete_reason | 删除原因 |
| deleted_at | 删除时间 |
| expires_at | 保留过期时间（删除时间 + 7天） |
| is_restored | 是否已恢复 |

**级联删除流程**：
- 删除台站时，其所有日志自动进入 `deleted_logs`（7 天保留），关联位置软删除，台站软删除
- 软删除的日志不进回收站（可通过直接逻辑删除恢复）
- 恢复日志时从 `log_data` JSON 还原，创建新的 QSO 记录

## SQLite 兼容说明

项目默认使用 SQLite 数据库。SQLite 与 MySQL 的差异处理包括：

- **连接参数**：`check_same_thread=False`（SQLite），`pool_pre_ping=True` + `pool_recycle=3600`（MySQL）
- **WAL模式**：SQLite 启动时自动执行 `PRAGMA journal_mode=WAL` 提升并发性能
- **外键**：SQLite 启动时自动执行 `PRAGMA foreign_keys=ON`
- **类型映射**：SQLAlchemy 自动处理 `INTEGER`/`VARCHAR`/`BOOLEAN`/`ENUM` 的类型差异
- **ENUM**：SQLite 不支持原生 ENUM，项目中使用 `VARCHAR` 替代

## 索引优化

### 关键索引
- `qso_logs`: (user_id, qso_date), (call_sign), (band), (mode)
- `users`: (username) UNIQUE, (email) UNIQUE
- `stations`: (user_id, callsign) UNIQUE
- `locations`: user_id, station_id

## 数据一致性

### 事务处理
- 日志导入使用事务，批量提交（每 500 条 commit 一次），避免事务过大超时
- 用户删除使用 CASCADE 保证数据完整性
- 位置激活使用事务确保全局唯一性（先取消所有位置激活，再激活目标位置）

### 级联删除策略
- **用户删除**：CASCADE 删除其台站、位置、日志
- **台站删除**：日志进回收站（7天保留）→ 位置软删除 → 台站软删除
- **位置删除**：软删除（`is_deleted=True`），日志的 `location_id` 设为 NULL（ON DELETE SET NULL）
- **日志删除**：软删除（`is_deleted=True`），查询时自动过滤

### 台站软删除恢复
- 删除台站后，若用户重新创建同名台站，系统自动恢复原台站记录而非创建新记录

## 备份建议

- **日备份**：每天凌晨执行完整备份
- **周备份**：每周备份保留至少4周
- **SQLite**：直接拷贝 `.db` 文件即可备份
