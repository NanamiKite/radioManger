# 部署说明文档

## 部署模式对比

| 特性 | 纯本地部署 | 局域网部署 | 云服务器部署 |
|-----|---------|---------|-----------|
| **前端** | Vite Web应用 | Vite Web应用 | Web应用 (Nginx) |
| **后端** | 必需（本地） | 必需 (Docker) | 必需 (Docker) |
| **数据库** | **SQLite** | MySQL (Docker) | MySQL (外部) |
| **部署位置** | 用户电脑 | 树莓派/NAS | 云服务器 |
| **网络访问** | 无 | 局域网 | 互联网 |
| **启动方式** | `start-local.sh` | `docker-compose up` | `docker-compose up` |
| **适用场景** | 个人、离线 | 小型俱乐部 | 生产、多用户 |

---

## 1. 纯本地部署（推荐）

使用 SQLite 数据库，无需安装 MySQL/Redis。

### 1.1 前置要求
- Python 3.10+
- Node.js 18+
- npm

### 1.2 一键启动（推荐）

```bash
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager
chmod +x start-local.sh
./start-local.sh
```

### 1.3 手动启动

```bash
# 后端
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库 (SQLite)
python -m app.scripts.init_db

# 启动后端
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端（新终端）
cd frontend
npm install
npm run dev
```

### 1.4 访问
- 前端: http://localhost:5173
- API: http://localhost:8000
- API文档: http://localhost:8000/docs
- 测试账号: `admin` / `admin123`

### 1.5 环境变量配置

编辑 `backend/.env`:

```env
DATABASE_MODE=sqlite              # 使用SQLite
SQLITE_URL=sqlite:///./radiomanager.db
SECRET_KEY=your-secret-key-here
ENABLE_TEST_ACCOUNT=true
DEBUG=true
```

## 2. 局域网部署（推荐树莓派）

### 2.1 前置要求
- 树莓派 4B+ / NAS / 任意Linux服务器
- Docker + Docker Compose

### 2.2 部署步骤

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 2. 配置环境变量
cp backend/.env.example backend/.env
# 编辑 .env，设置 DATABASE_MODE=mysql 和正确的连接信息

# 3. 启动Docker容器
docker-compose up -d

# 4. 初始化数据库
docker-compose exec backend python -m app.scripts.init_db
```

### 2.3 访问
- 前端: `http://<树莓派IP>:5173`
- API文档: `http://<树莓派IP>:8000/docs`

## 3. 云服务器部署

### 3.1 服务器配置
- 2核4GB 以上
- Ubuntu 20.04+
- Docker + Docker Compose

### 3.2 初始化

```bash
# 系统更新
apt update && apt upgrade -y

# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 克隆项目并部署
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 配置SSL（Let's Encrypt）
apt install certbot
certbot certonly --standalone -d radiomanager.example.com

# 编辑 .env，配置生产环境参数
# ...

# 启动
docker-compose up -d
docker-compose exec backend python -m app.scripts.init_db
```

### 3.3 Nginx 反向代理

```nginx
server {
    listen 443 ssl;
    server_name radiomanager.example.com;

    ssl_certificate /etc/letsencrypt/live/.../fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/.../privkey.pem;

    location / {
        proxy_pass http://frontend:5173;
    }
    location /api/ {
        proxy_pass http://backend:8000;
    }
}
```

## 4. 数据库初始化

```bash
# 创建所有表 + 管理员账号
cd backend
python -m app.scripts.init_db
```

脚本自动完成：
1. 创建9张数据表
2. 检测并创建管理员账号（默认 `admin / admin123`）
3. 输出数据库模式信息

### 手动创建管理员

```bash
python -m app.scripts.create_admin [username] [password] [email]
```

## 5. 启动脚本说明

### start-local.sh
启动 SQLite 本地环境，自动创建虚拟环境并初始化数据库。

```bash
# 流程
1. 创建 Python venv + 安装依赖
2. python -m app.scripts.init_db（初始化SQLite）
3. uvicorn app.main:app（后端）
4. npm install + npm run dev（前端）
```

### start-dev.sh
支持 `DATABASE_MODE` 环境变量，可选 SQLite 或 MySQL。

```bash
DATABASE_MODE=sqlite ./start-dev.sh   # SQLite（默认）
DATABASE_MODE=mysql ./start-dev.sh    # MySQL
```

## 6. 常见问题

### Q: MySQL 连接错误
确保 MySQL 服务运行，`.env` 中 `DATABASE_URL` 正确。

### Q: 前端无法连接后端
Vite 配置了 `/api` 代理到 `localhost:8000`。确认后端已启动。

### Q: 端口被占用
```bash
kill $(lsof -ti:8000)    # 释放后端端口
kill $(lsof -ti:5173)    # 释放前端端口
```

### Q: SQLite 文件位置
默认在当前工作目录生成 `radiomanager.db`。可通过 `SQLITE_URL` 配置路径。
