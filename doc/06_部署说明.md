# 部署说明文档

## 部署模式对比

| 特性 | 纯本地部署 | 局域网部署 | 云服务器部署 |
|-----|---------|---------|-----------|
| **前端** | Electron应用 | Electron/Web | Web应用 |
| **后端** | 可选 | 必需 | 必需 |
| **数据库** | SQLite/本地 | MySQL(Docker) | 云数据库 |
| **部署位置** | 用户电脑 | 树莓派/NAS | 云服务器 |
| **网络访问** | 无 | 局域网 | 互联网 |
| **维护成本** | 低 | 中 | 高 |
| **扩展性** | 低 | 中 | 高 |
| **适用场景** | 个人、离线 | 小型俱乐部 | 生产、多用户 |

## 1. 纯本地部署

### 1.1 前置要求
- Windows 10+、macOS 10.13+、Ubuntu 18.04+
- Node.js 16+
- Git

### 1.2 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 2. 进入前端目录
cd frontend

# 3. 安装依赖
npm install

# 4. 运行开发环境
npm run dev

# 或构建Electron应用
npm run electron-build
```

### 1.3 配置文件

创建 `frontend/.env.local`:
```env
VITE_APP_MODE=local
VITE_APP_API_MODE=local
```

### 1.4 数据存储

本地模式使用：
- **IndexedDB**：存储日志数据
- **localStorage**：存储用户设置
- **文件系统**：保存导出的ADI文件（Electron）

### 1.5 本地API开发

可选：运行本地后端服务
```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 运行服务
uvicorn app.main:app --reload
```

## 2. 局域网部署（推荐树莓派）

### 2.1 前置要求
- 树莓派 4B+ (4GB内存以上) 或 NAS
- 操作系统：Raspberry Pi OS、Ubuntu Server
- Docker和Docker Compose
- Git

### 2.2 硬件配置建议

```
树莓派4B配置：
- CPU: 4核
- 内存: 4GB或8GB
- 存储: 64GB+（SD卡或外接SSD）
- 网络: 有线以太网（推荐）或WiFi
- 电源: 5V/3A
```

### 2.3 系统安装

#### 2.3.1 Raspberry Pi OS安装
```bash
# 使用Raspberry Pi Imager安装：
1. 下载Raspberry Pi Imager
2. 选择Raspberry Pi OS（推荐64位版本）
3. 选择SD卡
4. 写入系统

# 系统初始化
sudo raspi-config
# 1. 扩展文件系统
# 2. 启用SSH
# 3. 更新系统
```

#### 2.3.2 Docker安装

```bash
# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 添加用户到docker组
sudo usermod -aG docker $USER

# 安装Docker Compose
sudo apt install docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 2.4 项目部署

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 2. 创建环境文件
cp backend/.env.example backend/.env

# 3. 编辑环境变量
nano backend/.env
```

编辑 `backend/.env`:
```env
# 应用设置
DEBUG=false
DATABASE_URL=mysql://radiomanager:password123@mysql:3306/radiomanager
REDIS_URL=redis://redis:6379

# JWT设置
SECRET_KEY=your-super-secret-key-change-this-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=10080

# 邮件设置
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-password
FROM_EMAIL=noreply@radiomanager.local

# 数据库初始密码
DB_ROOT_PASSWORD=rootpass123
DB_NAME=radiomanager
DB_USER=radiomanager
DB_PASSWORD=password123
```

```bash
# 4. 启动Docker Compose
docker-compose up -d

# 5. 初始化数据库
docker-compose exec backend alembic upgrade head

# 6. 创建管理员账户
docker-compose exec backend python -m app.scripts.create_admin
```

### 2.5 前端部署

```bash
# 1. 构建前端
cd frontend
npm install
npm run build

# 2. 使用Nginx提供静态文件
# 创建Dockerfile（如果没有）
```

创建 `frontend/Dockerfile`:
```dockerfile
FROM node:16-alpine as builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

更新 `docker-compose.yml` 添加前端：
```yaml
frontend:
  build:
    context: ./frontend
    dockerfile: Dockerfile
  ports:
    - "80:80"
  depends_on:
    - backend
  networks:
    - radiomanager
```

```bash
# 3. 重新启动服务
docker-compose down
docker-compose up -d
```

### 2.6 访问应用

在同一局域网的浏览器中访问：
- 前端：`http://<树莓派IP>`
- API文档：`http://<树莓派IP>:8000/docs`

### 2.7 定时备份配置

创建备份脚本 `backup.sh`:
```bash
#!/bin/bash

BACKUP_DIR="/home/pi/radiomanager-backups"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
docker-compose exec -T mysql mysqldump -u radiomanager -ppassword123 radiomanager > $BACKUP_DIR/db_$TIMESTAMP.sql

# 备份Redis
docker-compose exec -T redis redis-cli BGSAVE

# 压缩备份
tar -czf $BACKUP_DIR/backup_$TIMESTAMP.tar.gz -C $BACKUP_DIR *.sql

# 删除7天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR/backup_$TIMESTAMP.tar.gz"
```

```bash
# 添加到crontab（每天凌晨2点执行）
crontab -e
# 添加: 0 2 * * * /home/pi/radiomanager-backup.sh
```

## 3. 云服务器部署

### 3.1 服务器选择

推荐配置（以阿里云为例）：
- **轻量应用服务器**：2核4GB，40GB SSD
- **操作系统**：Ubuntu 20.04 LTS
- **公网带宽**：5Mbps

### 3.2 域名和SSL配置

```bash
# 1. 购买域名（阿里云、腾讯云等）

# 2. 配置DNS指向服务器IP

# 3. 生成SSL证书（使用Let's Encrypt）
sudo apt install certbot python3-certbot-nginx

certbot certonly --standalone -d radiomanager.example.com

# 4. 证书自动续期
sudo systemctl enable certbot.timer
sudo systemctl start certbot.timer
```

### 3.3 服务器初始化

```bash
# 1. SSH连接
ssh root@your_server_ip

# 2. 系统更新
apt update && apt upgrade -y

# 3. 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 4. 安装Docker Compose
apt install -y docker-compose

# 5. 配置防火墙
ufw enable
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 8000/tcp  # 临时，后期移除
```

### 3.4 项目部署

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 2. 配置环境变量
cp backend/.env.example backend/.env
nano backend/.env
```

编辑生产环境 `.env`:
```env
# 应用设置
DEBUG=false
WORKERS=4
DATABASE_URL=mysql://radiomanager:your_password@mysql:3306/radiomanager
REDIS_URL=redis://redis:6379

# 安全设置
SECRET_KEY=$(openssl rand -hex 32)  # 生成随机密钥
ALLOWED_HOSTS=radiomanager.example.com,www.radiomanager.example.com

# SSL/TLS
SSL_ENABLED=true
CERT_PATH=/etc/letsencrypt/live/radiomanager.example.com/fullchain.pem
KEY_PATH=/etc/letsencrypt/live/radiomanager.example.com/privkey.pem

# 数据库
DB_ROOT_PASSWORD=secure_password_123
DB_USER=radiomanager
DB_PASSWORD=secure_password_456

# 邮件（生产使用）
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=your-app-specific-password

# QRZ API
QRZ_USERNAME=your_qrz_username
QRZ_PASSWORD=your_qrz_password

# GitHub集成
GITHUB_TOKEN=your_github_token
```

### 3.5 Nginx反向代理配置

创建 `nginx/radiomanager.conf`:
```nginx
server {
    listen 80;
    server_name radiomanager.example.com www.radiomanager.example.com;
    
    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name radiomanager.example.com www.radiomanager.example.com;

    # SSL证书
    ssl_certificate /etc/letsencrypt/live/radiomanager.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/radiomanager.example.com/privkey.pem;
    
    # SSL配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 前端静态文件
    location / {
        proxy_pass http://frontend:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端API
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 不缓存API响应
    location /api/ {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }

    # 缓存静态资源
    location /static/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 3.6 更新docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./backups:/backups
    networks:
      - radiomanager
    command: --default-authentication-plugin=mysql_native_password

  redis:
    image: redis:7-alpine
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - radiomanager

  backend:
    build: ./backend
    restart: always
    environment:
      - DATABASE_URL=${DATABASE_URL}
      - REDIS_URL=${REDIS_URL}
      - SECRET_KEY=${SECRET_KEY}
    depends_on:
      - mysql
      - redis
    networks:
      - radiomanager
    expose:
      - 8000

  frontend:
    build: ./frontend
    restart: always
    networks:
      - radiomanager
    expose:
      - 80

  nginx:
    image: nginx:alpine
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/radiomanager.conf:/etc/nginx/conf.d/default.conf
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - frontend
      - backend
    networks:
      - radiomanager

volumes:
  mysql_data:
  redis_data:

networks:
  radiomanager:
    driver: bridge
```

### 3.9 自动备份脚本

创建 `/usr/local/bin/radiomanager-backup.sh`:
```bash
#!/bin/bash

BACKUP_DIR="/var/radiomanager/backups"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份数据库
docker exec radiomanager-mysql-1 mysqldump -u radiomanager -ppassword123 radiomanager | gzip > $BACKUP_DIR/db_$TIMESTAMP.sql.gz

# 删除30天前的备份
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +30 -delete

echo "Backup completed: $BACKUP_DIR/db_$TIMESTAMP.sql.gz"
```

```bash
# 添加执行权限
chmod +x /usr/local/bin/radiomanager-backup.sh

# 添加到crontab（每天凌晨3点执行）
crontab -e
# 0 3 * * * /usr/local/bin/radiomanager-backup.sh
```

## 5. 故障恢复

### 5.1 常见问题排查

```bash
# 1. 检查容器状态
docker-compose ps

# 2. 查看容器日志
docker-compose logs backend

# 3. 重启服务
docker-compose restart backend

# 4. 检查网络连接
docker-compose exec backend ping mysql

# 5. 检查磁盘空间
df -h
```

### 5.2 数据库恢复

```bash
# 1. 恢复备份
docker-compose exec mysql mysql -u radiomanager -ppassword123 radiomanager < backup.sql

# 2. 验证恢复
docker-compose exec mysql mysql -u radiomanager -ppassword123 radiomanager -e "SELECT COUNT(*) FROM qso_logs;"
```

## 7. 常用命令速查

```bash
# Docker相关
docker-compose up -d              # 启动所有服务
docker-compose down               # 停止所有服务
docker-compose logs -f            # 查看实时日志
docker-compose exec mysql mysql   # 连接数据库
docker ps                         # 查看运行的容器

# 数据库相关
alembic revision -m "description" # 创建新的迁移
alembic upgrade head              # 应用所有迁移
alembic downgrade -1              # 回滚最后一次迁移

# 备份相关
mysqldump -u user -p db > backup.sql  # 备份数据库
mysql -u user -p db < backup.sql      # 恢复数据库

# 系统相关
df -h                             # 查看磁盘使用
free -h                           # 查看内存使用
top                               # 查看进程状态
systemctl status service-name     # 查看服务状态
```
