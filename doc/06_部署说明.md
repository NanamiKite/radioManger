# 部署说明文档

## 部署模式对比

| 特性 | 纯本地部署 | 局域网部署 | 云服务器部署 | Windows桌面 | Android移动 |
|-----|---------|---------|-----------|------------|-----------|
| **前端** | Electron/Vite | Web应用 | Nginx | **Electron** | **Capacitor/APK** |
| **后端** | 必需（本地） | 必需 (Docker) | 必需 (Docker) | 同左/远程 | 同左/远程 |
| **数据库** | **SQLite** | MySQL (Docker) | MySQL (外部) | SQLite/远程 | 远程 |
| **适用场景** | 个人、离线 | 小型俱乐部 | 生产、多用户 | Windows桌面 | 手机浏览器 |
| **启动方式** | `start-local.sh` | `docker-compose up` | `docker-compose up` | `start-windows-local.bat` | PWA/APK |

---

## 1. 纯本地部署（推荐）

使用 SQLite 数据库，无需安装 MySQL/Redis。

### 1.1 前置要求
- Python 3.10+
- Node.js 18+
- npm

### 1.2 一键启动（推荐）

```bash
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager
chmod +x start-local.sh
./start-local.sh
```

### 1.3 手动启动

```bash
# 后端
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库 (SQLite)
python -m app.scripts.init_db

# 启动后端
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端（新终端）
cd frontend
npm install
npm run dev
```

### 1.4 访问
- 前端: http://localhost:5173
- API: http://localhost:8000
- API文档: http://localhost:8000/docs
- 测试账号: `admin` / `admin123`

### 1.5 环境变量配置

编辑 `backend/.env`:

```env
DATABASE_MODE=sqlite              # 使用SQLite
SQLITE_URL=sqlite:///./radiomanager.db
SECRET_KEY=your-secret-key-here
ENABLE_TEST_ACCOUNT=true
DEBUG=true
```

## 2. 局域网部署

适用于小型俱乐部、HAM 活动现场等场景。提供两种方案：Docker 全栈部署和前后端分机部署。

### 2.1 方案 A：Docker 全栈部署（推荐）

前后端 + 数据库运行在同一台服务器上，通过 Docker Compose 统一管理。

**前置要求**：
- 任意 Linux 服务器（树莓派 4B+ / NAS / PC）
- Docker + Docker Compose

**部署步骤**：

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radioManger

# 2. 配置环境变量
cp backend/.env.example backend/.env
```

编辑 `backend/.env`，关键配置：

```env
# 数据库：Docker 部署使用 MySQL
DATABASE_MODE=mysql
DATABASE_URL=mysql+pymysql://radiomanager:password123@mysql:3306/radiomanager

# JWT 密钥（务必修改）
SECRET_KEY=your-random-secret-key-here

# CORS：添加服务器 IP（前端通过浏览器访问的地址）
CORS_ORIGINS=http://192.168.1.100:5173,http://localhost:5173

# 关闭调试模式
DEBUG=false
```

```bash
# 3. 启动所有容器
docker-compose up -d

# 4. 初始化数据库（首次部署时执行）
docker-compose exec backend python -m app.scripts.init_db
```

**访问**：
- 前端: `http://<服务器IP>:5173`
- API 文档: `http://<服务器IP>:8000/docs`
- 测试账号: `admin` / `admin123`

**容器说明**（`docker-compose.yml` 启动 4 个服务）：

| 服务 | 端口 | 说明 |
|------|------|------|
| `mysql` | 3306 | MySQL 8.0 数据库 |
| `redis` | 6379 | Redis 缓存（预留） |
| `backend` | 8000 | FastAPI 后端 |
| `frontend` | 5173 | Vite 开发服务器 |

> **生产环境提示**：`docker-compose.yml` 中的数据库密码和 `SECRET_KEY` 必须修改。前端服务仅用于开发，生产环境应使用 Nginx 托管前端静态文件（见第 3 节）。

---

### 2.2 方案 B：前后端分机部署

前端和后端分别部署在不同机器上。例如：后端运行在实验室服务器，前端在个人笔记本上开发。

**网络拓扑**：
```
[前端机器 192.168.1.50]  ──HTTP──▶  [后端机器 192.168.1.100:8000]
     Vite :5173                          FastAPI + SQLite/MySQL
```

#### 后端机器（192.168.1.100）

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radioManger/backend

# 2. 创建虚拟环境并安装依赖
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
```

编辑 `backend/.env`：

```env
# 数据库：根据需求选择
DATABASE_MODE=sqlite
SQLITE_URL=sqlite:///./radiomanager.db
# 或 MySQL：
# DATABASE_MODE=mysql
# DATABASE_URL=mysql+pymysql://user:pass@localhost:3306/radiomanager

# JWT 密钥（务必修改）
SECRET_KEY=your-random-secret-key-here

# CORS：必须添加前端机器的地址！
CORS_ORIGINS=http://192.168.1.50:5173,http://localhost:5173

# 允许外部访问
DEBUG=false
```

```bash
# 4. 初始化数据库
python -m app.scripts.init_db

# 5. 启动后端（监听所有网卡）
source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

> **防火墙**：确保后端机器的 8000 端口已开放：`sudo ufw allow 8000`

#### 前端机器（192.168.1.50）

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radioManger/frontend

# 2. 安装依赖
npm install

# 3. 修改 API 代理地址
```

编辑 `frontend/vite.config.ts`，将 proxy 的 `target` 改为后端机器地址：

```typescript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://192.168.1.100:8000',  // ← 改为后端机器 IP
      changeOrigin: true,
      ws: true  // WebSocket 转发（DX Cluster 需要）
    }
  }
}
```

```bash
# 4. 启动前端
npm run dev
```

**访问**：浏览器打开 `http://localhost:5173`

#### 关键配置对照

| 配置项 | 后端机器 | 前端机器 |
|--------|---------|---------|
| `backend/.env` → `CORS_ORIGINS` | 必须包含 `http://前端IP:5173` | — |
| `frontend/vite.config.ts` → `proxy.target` | — | `http://后端IP:8000` |
| 防火墙 | 开放 8000 端口 | 开放 5173 端口（如需局域网访问） |

> **WebSocket 说明**：DX Cluster 实时 spot 功能依赖 WebSocket 连接。`vite.config.ts` 中的 `ws: true` 确保开发模式下 WebSocket 请求被正确代理到后端。

---

### 2.3 数据库选择建议

| 场景 | 推荐数据库 | 配置 |
|------|-----------|------|
| 单人使用、离线环境 | SQLite（默认） | `DATABASE_MODE=sqlite` |
| 多人共享、局域网服务器 | MySQL（Docker） | `DATABASE_MODE=mysql` |
| 开发/测试 | SQLite | 无需额外安装 |

SQLite 无需安装数据库服务，数据存储在单个 `.db` 文件中，备份只需拷贝该文件。MySQL 适合多用户并发访问场景。

## 3. 云服务器部署

适用于多用户生产环境，前端静态文件由 Nginx 托管，后端通过反向代理暴露。

### 3.1 服务器配置要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 1 核 | 2 核+ |
| 内存 | 1 GB | 4 GB+ |
| 磁盘 | 20 GB | 50 GB+ |
| 系统 | Ubuntu 20.04+ | Ubuntu 22.04 LTS |
| 软件 | Docker + Docker Compose | 同左 |

### 3.2 部署步骤

#### 步骤 1：安装基础软件

```bash
# 系统更新
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# 安装 Nginx（用于反向代理和前端托管）
sudo apt install -y nginx certbot python3-certbot-nginx

# 安装 Git
sudo apt install -y git
```

#### 步骤 2：克隆项目并配置

```bash
git clone https://github.com/NanamiKite/radioManger.git
cd radioManger
```

编辑 `backend/.env`：

```env
# 数据库（云服务器推荐 MySQL）
DATABASE_MODE=mysql
DATABASE_URL=mysql+pymysql://radiomanager:YOUR_DB_PASSWORD@mysql:3306/radiomanager

# JWT 密钥（必须使用随机长字符串）
SECRET_KEY=your-random-32-char-secret-key-here

# CORS：添加你的域名
CORS_ORIGINS=https://radiomanager.example.com,http://localhost:5173

# 关闭调试
DEBUG=false
ENVIRONMENT=production

# 日志
LOG_LEVEL=INFO
LOG_FILE=logs/app.log
```

#### 步骤 3：构建前端静态文件

```bash
cd frontend
npm install
npm run build
# 产物在 frontend/dist/ 目录
cd ..
```

#### 步骤 4：启动后端

```bash
# 方式一：Docker（推荐）
docker-compose up -d mysql backend
docker-compose exec backend python -m app.scripts.init_db

# 方式二：直接运行
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m app.scripts.init_db
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4 &
```

#### 步骤 5：配置 Nginx

```bash
# 将前端构建产物拷贝到 Nginx 目录
sudo mkdir -p /var/www/radiomanager
sudo cp -r frontend/dist/* /var/www/radiomanager/

# 配置 Nginx
sudo nano /etc/nginx/sites-available/radiomanager
```

写入以下配置：

```nginx
# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name radiomanager.example.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl http2;
    server_name radiomanager.example.com;

    # ── SSL 证书 ──
    ssl_certificate     /etc/letsencrypt/live/radiomanager.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/radiomanager.example.com/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # ── 安全头 ──
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "SAMEORIGIN" always;

    # ── 前端静态文件 ──
    root /var/www/radiomanager;
    index index.html;

    # SPA 路由：所有非文件请求回退到 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # ── 后端 API 反向代理 ──
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_request_buffering off;
        proxy_buffering off;
    }

    # ── WebSocket 代理（DX Cluster 实时 spot） ──
    location /api/v1/dxcluster/ws {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }

    # ── 压缩 ──
    gzip on;
    gzip_types text/plain text/css text/javascript application/json application/javascript;
    gzip_min_length 1024;
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/radiomanager /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default  # 移除默认配置

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

#### 步骤 6：申请 SSL 证书

```bash
# 使用 Let's Encrypt 自动申请（需域名已解析到服务器 IP）
sudo certbot --nginx -d radiomanager.example.com

# 自动续期测试
sudo certbot renew --dry-run
```

#### 步骤 7：开放防火墙

```bash
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 22/tcp    # SSH
sudo ufw enable
```

### 3.3 访问

- 前端: `https://radiomanager.example.com`
- API 文档: `https://radiomanager.example.com/api/v1/docs`（需在 Nginx 中额外代理 `/docs` 路径，或临时开放 8000 端口访问 `http://服务器IP:8000/docs`）
- 测试账号: `admin` / `admin123`

### 3.4 生产环境安全清单

| 项目 | 操作 |
|------|------|
| `SECRET_KEY` | 使用 `openssl rand -hex 32` 生成随机密钥 |
| 数据库密码 | 修改 `docker-compose.yml` 和 `.env` 中的默认密码 |
| 测试账号 | 首次登录后修改 `admin` 密码或禁用 `ENABLE_TEST_ACCOUNT` |
| CORS | 仅允许你的域名，不要使用 `*` |
| 端口 | 仅开放 80/443/22，不暴露 8000/3306/6379 |
| 日志 | 定期检查 `backend/logs/app.log` |

## 5. Android 部署

```bash
cd frontend
npm install
```

```bash
# 2. 初始化 Capacitor (仅首次)
npx cap init RadioManager com.radiomanager.app

# 4. 构建 APK (需要 Android Studio 或 Gradle)
```

## 8. 常见问题

### Q: 前端无法连接后端（前后端同机）

Vite 开发模式下，`vite.config.ts` 配置了 `/api` 代理到 `localhost:8000`。确认后端已启动：

```bash
curl http://localhost:8000/health
# 应返回: {"status":"ok","database":"sqlite"}
```

### Q: 前端无法连接后端（前后端不同机）

**排查步骤**：

1. **确认后端可达**：从前端机器测试
   ```bash
   curl http://<后端IP>:8000/health
   ```

2. **检查 `vite.config.ts` 的 proxy target**：是否改为后端机器 IP
   ```typescript
   proxy: {
     '/api': {
       target: 'http://192.168.1.100:8000',  // ← 必须是后端机器 IP
       changeOrigin: true,
       ws: true
     }
   }
   ```

3. **检查后端防火墙**：8000 端口是否开放
   ```bash
   sudo ufw status
   sudo ufw allow 8000/tcp
   ```

4. **检查后端启动参数**：必须监听所有网卡
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000
   # 不要用 --host 127.0.0.1（只监听本地）
   ```

### Q: CORS 跨域错误

浏览器控制台报 `Access-Control-Allow-Origin` 错误时：

1. 编辑 `backend/.env`，在 `CORS_ORIGINS` 中添加前端地址：
   ```env
   CORS_ORIGINS=http://192.168.1.50:5173,http://localhost:5173
   ```
2. 重启后端服务
3. 多个地址用逗号分隔，不要有空格

### Q: WebSocket 连接失败（DX Cluster）

DX Cluster 实时 spot 功能依赖 WebSocket。排查：

1. **开发模式**：确认 `vite.config.ts` 中 `ws: true`
2. **Nginx 部署**：确认配置了 WebSocket 代理头：
   ```nginx
   location /api/v1/dxcluster/ws {
       proxy_http_version 1.1;
       proxy_set_header Upgrade $http_upgrade;
       proxy_set_header Connection "upgrade";
   }
   ```
3. **检查后端日志**：WebSocket 连接失败会有错误日志

### Q: MySQL 连接错误

确保 MySQL 服务运行，`.env` 中 `DATABASE_URL` 正确：

```bash
# Docker 环境：检查容器状态
docker-compose ps
docker-compose logs mysql

# 直接连接测试
mysql -u radiomanager -p -h 127.0.0.1 radiomanager
```

### Q: 端口被占用

```bash
# 查看占用端口的进程
lsof -i:8000
lsof -i:5173

# 终止进程
kill $(lsof -ti:8000)
kill $(lsof -ti:5173)
```

### Q: SQLite 文件位置

默认在后端工作目录生成 `radiomanager.db`。可通过 `SQLITE_URL` 配置路径：

```env
SQLITE_URL=sqlite:///./radiomanager.db          # 当前目录
SQLITE_URL=sqlite:////data/radiomanager.db      # 绝对路径
```

### Q: 生产环境前端刷新 404

Nginx 配置中必须添加 SPA 回退规则：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

没有此规则时，直接访问 `/logs/123` 等前端路由会返回 404。
