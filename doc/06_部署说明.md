# 部署说明文档

## 部署模式对比

| 特性 | 纯本地部署 | 局域网部署 | 云服务器部署 | Windows桌面 | Android移动 |
|-----|---------|---------|-----------|------------|-----------|
| **前端** | Electron/Vite | Web应用 | Nginx | **Electron** | **Capacitor/APK** |
| **后端** | 必需（本地） | 必需 (Docker) | 必需 (Docker) | 同左/远程 | 同左/远程 |
| **数据库** | **SQLite** | MySQL (Docker) | MySQL (外部) | SQLite/远程 | 远程 |
| **适用场景** | 个人、离线 | 小型俱乐部 | 生产、多用户 | Windows桌面 | 手机浏览器 |
| **启动方式** | `start-local.sh` | `docker-compose up` | `docker-compose up` | `start-windows-local.bat` | PWA/APK |

---

## 1. 纯本地部署（推荐）

使用 SQLite 数据库，无需安装 MySQL/Redis。

### 1.1 前置要求
- Python 3.10+
- Node.js 18+
- npm

### 1.2 一键启动（推荐）

```bash
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager
chmod +x start-local.sh
./start-local.sh
```

### 1.3 手动启动

```bash
# 后端
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库 (SQLite)
python -m app.scripts.init_db

# 启动后端
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 前端（新终端）
cd frontend
npm install
npm run dev
```

### 1.4 访问
- 前端: http://localhost:5173
- API: http://localhost:8000
- API文档: http://localhost:8000/docs
- 测试账号: `admin` / `admin123`

### 1.5 环境变量配置

编辑 `backend/.env`:

```env
DATABASE_MODE=sqlite              # 使用SQLite
SQLITE_URL=sqlite:///./radiomanager.db
SECRET_KEY=your-secret-key-here
ENABLE_TEST_ACCOUNT=true
DEBUG=true
```

## 2. 局域网部署（推荐树莓派）

### 2.1 前置要求
- 树莓派 4B+ / NAS / 任意Linux服务器
- Docker + Docker Compose

### 2.2 部署步骤

```bash
# 1. 克隆项目
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 2. 配置环境变量
cp backend/.env.example backend/.env
# 编辑 .env，设置 DATABASE_MODE=mysql 和正确的连接信息

# 3. 启动Docker容器
docker-compose up -d

# 4. 初始化数据库
docker-compose exec backend python -m app.scripts.init_db
```

### 2.3 访问
- 前端: `http://<树莓派IP>:5173`
- API文档: `http://<树莓派IP>:8000/docs`

## 3. 云服务器部署

### 3.1 服务器配置
- 2核4GB 以上
- Ubuntu 20.04+
- Docker + Docker Compose

### 3.2 初始化

```bash
# 系统更新
apt update && apt upgrade -y

# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 克隆项目并部署
git clone https://github.com/NanamiKite/radioManger.git
cd radiomanager

# 配置SSL（Let's Encrypt）
apt install certbot
certbot certonly --standalone -d radiomanager.example.com

# 编辑 .env，配置生产环境参数
# ...

# 启动
docker-compose up -d
docker-compose exec backend python -m app.scripts.init_db
```

### 3.3 Nginx 反向代理

```nginx
server {
    listen 443 ssl;
    server_name radiomanager.example.com;

    ssl_certificate /etc/letsencrypt/live/.../fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/.../privkey.pem;

    location / {
        proxy_pass http://frontend:5173;
    }
    location /api/ {
        proxy_pass http://backend:8000;
    }
}
```

## 4. Windows 部署

### 4.1 纯本地部署 (Electron 桌面应用 + SQLite)

在 Windows 上运行完整的 RadioManager（后端 + 前端桌面应用）。

**前置要求**：
- Python 3.10+
- Node.js 18+
- Git

**一键启动**：
```batch
双击 start-windows-local.bat
```

脚本自动完成：
1. 创建 Python 虚拟环境 + 安装依赖
2. 初始化 SQLite 数据库
3. 启动后端服务 (localhost:8000)
4. 启动 Electron 桌面应用

### 4.2 Web 前端部署 (连接远程后端)

在 Windows 上仅运行前端，连接局域网或云服务器的后端。

```batch
双击 start-windows-web.bat
# 选择:
#   [1] 局域网模式 - 连接局域网后端
#   [2] 云服务器模式 - 连接云端后端
#   [3] Electron桌面应用
```

### 4.3 Electron 打包 (Windows 安装包)

```bash
cd frontend
npm install

# 构建 Windows 安装程序 (NSIS)
npm run electron:win

# 构建 Windows 便携版 (免安装)
npm run electron:win-portable
```

产物在 `frontend/release/` 目录：
- `RadioManager Setup {version}.exe` — 安装程序
- `RadioManager-{version}-portable.exe` — 便携版

## 5. Android 部署

### 5.1 PWA 方式 (推荐)

构建完成后浏览器打开前端，自动支持 PWA 安装：

```bash
cd frontend
npm install
npm run build
npm run preview
```

Android 用户通过 Chrome 打开 → 菜单 "添加到主屏幕" → 作为独立应用运行。

### 5.2 Capacitor APK 打包

```bash
cd frontend
npm install

# 1. 构建前端
npm run build

# 2. 初始化 Capacitor (仅首次)
npx cap init RadioManager com.radiomanager.app
npx cap add android

# 3. 同步前端代码到 Android 项目
npm run android:sync

# 4. 构建 APK (需要 Android Studio 或 Gradle)
npm run android:build

# APK 位置: frontend/android/app/build/outputs/apk/debug/app-debug.apk
```

> **注意**: Capacitor APK 为纯前端 WebView 封装，适用于纯本地（连接 PC 后端）和云/局域网部署。后端不内置于 APK 中。

### 5.3 PWA 配置

项目已内置 PWA 支持：
- `frontend/public/manifest.json` — PWA 清单
- `frontend/index.html` — 已配置 manifest + 主题色 + meta 标签
- Service Worker (可选) — 需自行添加

## 7. 启动脚本说明

### start-local.sh
启动 SQLite 本地环境，自动创建虚拟环境并初始化数据库。

```bash
# 流程
1. 创建 Python venv + 安装依赖
2. python -m app.scripts.init_db（初始化SQLite）
3. uvicorn app.main:app（后端）
4. npm install + npm run dev（前端）
```

### start-dev.sh
支持 `DATABASE_MODE` 环境变量，可选 SQLite 或 MySQL。

```bash
DATABASE_MODE=sqlite ./start-dev.sh   # SQLite（默认）
DATABASE_MODE=mysql ./start-dev.sh    # MySQL
```

## 8. 常见问题

### Q: MySQL 连接错误
确保 MySQL 服务运行，`.env` 中 `DATABASE_URL` 正确。

### Q: 前端无法连接后端
Vite 配置了 `/api` 代理到 `localhost:8000`。确认后端已启动。

### Q: 端口被占用
```bash
kill $(lsof -ti:8000)    # 释放后端端口
kill $(lsof -ti:5173)    # 释放前端端口
```

### Q: SQLite 文件位置
默认在当前工作目录生成 `radiomanager.db`。可通过 `SQLITE_URL` 配置路径。
